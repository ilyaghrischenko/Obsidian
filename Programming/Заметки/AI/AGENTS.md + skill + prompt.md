## Введение
Максимальная эффективность кодогенерации достигается только при правильном слиянии трех уровней управления агентом: `AGENTS.md`, глобального `SKILL.md` и пользовательского промпта. Эта связка (триада) исключает галлюцинации и позволяет ИИ писать компилируемый production-ready код с первого раза.

## Как это работает?
Каждый элемент в этой связке имеет строгую и изолированную зону ответственности:

1. **`AGENTS.md` (Контекст / Законы стройплощадки):**
   - Отвечает за **СПЕЦИФИКУ ПРОЕКТА**.
   - Сообщает ИИ: "Мы в проекте LudaFit, используем SQLite, возвращаем `Result<T>`, пишем Minimal API и не трекаем сущности при чтении".
2. **`SKILL.md` (Шаблон / Чертеж):**
   - Отвечает за **ПАТТЕРН И ФОРМУ**.
   - Сообщает ИИ: "Независимо от проекта, вертикальный слайс всегда пишется в один статический файл, строго без MediatR, с четким циклом самопроверки через компилятор".
3. **Prompt (Задача / Приказ на день):**
   - Отвечает за **БИЗНЕС-ЛОГИКУ**.
   - Сообщает ИИ: "Что конкретно нужно сделать прямо сейчас".

✅ Пример идеального Workflow в терминале:
```bash
# 1. Переходим в директорию, где лежит AGENTS.md (контекст загружается автоматически)
cd backend/LudaFit.Server/LudaFit.Domain/Features/Users

# 2. Вызываем глобальный скилл (шаблон) и передаем промпт (задачу)
$vertical-slice Создай фичу ApproveSignUpApplication. Принимает ID заявки, меняет статус на Approved, возвращает 204 NoContent.
```

## Итог

- **SKILL** говорит _КАК_ сделать.
    
- **AGENTS.md** говорит _В КАКОЙ СРЕДЕ_ и с какими библиотеками этот код будет работать.
    
- **Prompt** говорит _ЧТО_ именно нужно запрограммировать.

Такое разделение позволяет писать скиллы один раз и использовать их годами в разных монолитах, меняя только локальный `AGENTS.md`.
